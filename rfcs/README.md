# Request for Comments (RFC)

This directory contains design proposals and architectural decisions for ChickenBehaviorLab.

RFCs document the motivation, design rationale, alternatives, and expected impact of major technical decisions before they become part of the official project standards.

Once an RFC is accepted, its specifications are incorporated into the official documentation located under `docs/standards/`.

---

## RFC Lifecycle

1. Draft
2. Review
3. Accepted
4. Implemented
5. Archived (if superseded)

---

## Numbering

RFC documents use sequential numbering.

Example:

* RFC-0001: Chicken Behavior Annotation Standard
* RFC-0002: Chicken Behavior Ontology
* RFC-0003: Skeleton Representation
* RFC-0004: Behavior Event Specification

---

## Goals

RFCs provide:

* transparent decision making
* documented design rationale
* discussion of alternatives
* long-term project history
* reproducible architectural evolution

---

## Relationship with Official Standards

RFCs are proposals.

The official specifications are maintained in the `docs/standards/` directory.

Accepted RFCs eventually become part of the official project documentation.
