# Track Data Model

**Project:** ChickenBehaviorLab
**Document Type:** Canonical Data Model Specification
**Entity:** Track
**Version:** 1.0.0

---

# 1. Overview

The **Track** data model represents the continuous temporal identity of a chicken across multiple video frames.

Unlike a Detection, which describes an object in a single frame, a Track links multiple detections into a coherent trajectory belonging to the same individual.

The Track model is the foundation for pose estimation, behavior recognition, event generation, and long-term behavioral analysis.

---

# 2. Purpose

The purpose of the Track model is to maintain a stable identity for each detected chicken throughout a video sequence.

This temporal continuity enables downstream modules to analyze movement patterns, behavioral changes, and long-term activities.

---

# 3. Relationships

```text
Video
   │
   ├── Frame
   │      │
   │      └── Detection
   │
   └──────────────► Track
                        │
                        ├── Pose Sequence
                        ├── Skeleton Sequence
                        ├── Behavior Sequence
                        └── Behavior Events
```

A Track references many detections collected over time.

---

# 4. Required Attributes

| Field       | Type            | Description                              |
| ----------- | --------------- | ---------------------------------------- |
| track_id    | String          | Unique identifier of the tracked chicken |
| video_id    | String          | Source video identifier                  |
| first_frame | Integer         | First observed frame                     |
| last_frame  | Integer         | Last observed frame                      |
| detections  | List[Detection] | Ordered detection sequence               |
| trajectory  | List[Point]     | Center-point trajectory                  |

---

# 5. Optional Attributes

| Field            | Type    | Description                                |
| ---------------- | ------- | ------------------------------------------ |
| average_speed    | Float   | Estimated average speed                    |
| total_distance   | Float   | Travel distance                            |
| visibility_ratio | Float   | Fraction of visible frames                 |
| occlusion_count  | Integer | Number of occlusion events                 |
| lost_frames      | Integer | Frames where identity was temporarily lost |
| notes            | String  | Additional information                     |

---

# 6. Validation Rules

A valid Track shall satisfy:

* `track_id` must be unique within a video.
* `first_frame ≤ last_frame`.
* Detections must be ordered chronologically.
* All detections must belong to the same video.
* Frame numbers must be monotonically increasing.

---

# 7. Produced By

Typical tracking algorithms include:

* ByteTrack
* BoT-SORT
* DeepSORT
* OC-SORT
* Future multi-object tracking methods

The internal tracking algorithm does not affect the Track specification.

---

# 8. Consumed By

The Track model serves as input for:

* Pose Estimation
* Skeleton Generation
* Motion Analysis
* Behavior Recognition
* Visualization
* Behavioral Statistics

---

# 10. Design Decisions

The Track model contains only identity and trajectory information.

It intentionally excludes:

* pose estimates
* skeleton representations
* behavior labels
* health indicators

These are generated by independent downstream modules.

---

# 11. Future Extensions

Future versions may include:

* multi-camera identity association
* re-identification confidence
* trajectory uncertainty
* social interaction links
* spatial zone occupancy
* trajectory embeddings

---

# 12. Python Reference Model

Future implementations should provide an equivalent Python representation, for example:

```python
Track(
    track_id: str,
    video_id: str,
    first_frame: int,
    last_frame: int,
    detections: list[Detection],
    trajectory: list[Point]
)
```

This specification defines the canonical temporal representation of an individual chicken throughout the ChickenBehaviorLab pipeline.
